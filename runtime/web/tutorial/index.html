Tutorial Surface

Use the tool without guessing.

This page is intentionally narrow. It explains the tutorial flow, shows the current mode and roots, and gives the exact commands to prove, observe, and move into tool-dev safely.

Tutorial Only
1

Check where the tool thinks it lives

Always start by making the mode and roots explicit.

ai-workflow mode status --json
2

Prove against a real project

In tool-dev, the toolkit stays the repair target and the copied project is only evidence.

ai-workflow run ticket-proving-run --mode tool-dev --root adventure-machine2-playground --limit 5 --json
3

Inspect a single ticket without mutating anything

Use a dry run before you trust the executor.

ai-workflow run execution-dry-run --ticket REF-APP-SHELL-01 --json
4

Tell the tool it did bad

The observation flow now infers likely fields and auto-attaches the latest run artifact.

ai-workflow tool observe --mode tool-dev --complaint "it lied about readiness" --expected "it should admit the plan is weak" --create-ticket --json
