01. Core Philosophy

🛠️ Pessimistic Engineering

We treat LLMs as untrusted contributors. Every edit is a transaction. Every change is a patch. Every merge is verified.

📉 Token Efficiency

Context is currency. Our Graph-Powered Context Packer eliminates the "dump everything into prompt" anti-pattern, saving 80% on token costs.

02. Architecture

The Nervous System (SQLite)

All project state—symbols, dependencies, Kanban tickets, and metrics—lives in a local .ai-workflow/workflow.db file. This ensures the AI always has a perfect, non-hallucinated map of your project.

The Brain (Router)

Our 2D Competency Matrix matches tasks to the cheapest capable model. No more using GPT-4 for simple CSS tweaks.

03. Command Reference

ai-workflow shell "sweep bugs"

Triggers the autonomous fixer loop. Supergit creates a temp branch, applies patches, runs tests, and merges only on success.

ai-workflow init --brief path/to/project-brief.md

Starts a new project, normalizes the brief into a living document, and asks clarifying questions before epics are generated.

ai-workflow onboard path/to/project-brief.md

The Project Brief Onboarding flow. It turns a messy project description into a structured brief, then scopes epics and tickets.

ai-workflow consult

Handles the asynchronous architectural Q&A loop. Your guidance is indexed and used to train the system's local model.
